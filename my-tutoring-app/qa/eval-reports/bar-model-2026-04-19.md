## Notes — Fixes applied 2026-04-19

**Orchestrator refactor (SP-14 / SP-3):** Replaced single multi-mode generator with 6 per-mode sub-generators, each with a focused flat-slot schema where every rendered field is in `required`. Structurally eliminates the class of bugs where flash-lite silently drops nullable fields.

### Fixed issues

- **BM-1 (CRITICAL, read_scale missing options):** `options` is now derived deterministically in post-process from `expectedValue` and `scale.step` (4 distinct non-negative integers including expected). Gemini no longer supplies this field.
- **BM-2 (CRITICAL, read_scale answer leak via graphStyle='bar'):** `graphStyle` is now hardcoded per sub-generator (never trusted from Gemini). read_scale always emits `'scaled_bar'`. `targetBarIndex` derived via `findBarIndex(bars, targetBarLabel, expectedValue)` — Gemini supplies a label string, generator resolves to index. `scale` object synthesized deterministically from max bar value and step.
- **BM-3 (CRITICAL, scaled_bar_graph missing options):** Same fix as BM-1.

### Architectural changes

- Each sub-generator has fixed bar count (compare=2, read_scale=4, picture=4, scaled=5, word_problem=4, build=4) to avoid variable-length array quirks with flash-lite.
- Target bar is located by `targetBarLabel` (string) rather than `targetBarIndex` (number) — LLMs are unreliable at array indexing but reliable at echoing a label string.
- Picture graph `iconValue` forced to 2 or 5; all bar values snapped to nearest non-zero multiple of iconValue in post-process.
- Compare bars: if Gemini returns a `tallerBarLabel` that disagrees with the computed taller bar, the computed index wins.
- Options use `deriveOptions(expected, step)` — spacing by step keeps distractors on plausible axis values.

### Verification

- TypeScript: compiles cleanly (no new errors in gemini-bar-model.ts).
- Eval-test: 6/6 modes passing (was 4/6).
- G1 (required fields): all six modes produce every field the component reads.
- G4 (answer derivability): verified manually — expectedValue matches target bar's value; options include expectedValue; compare_bars targetBarIndex matches taller bar; graph_word_problem expectedValue matches arithmetic on bars.
