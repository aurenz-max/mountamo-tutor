## Notes — Fixed 2026-04-19

### DP-1 — `challenges` omission by flash-lite → fallback boilerplate
**Fix:** ORCHESTRATOR-REFACTOR. Replaced the single mega-generator with 6 per-mode sub-generators in [gemini-dot-plot.ts](../../src/components/lumina/service/math/gemini-dot-plot.ts). Each has a focused schema with all required fields (no nullable, no multi-purpose fields). Eliminates SP-14 and the "`challenges` absent from `required`" gap by construction — each mode's schema owns only the scalar challenge fields (`instruction`, `hint`, `narration`, mode-specific answer) so flash-lite cannot drop them. `range` replaced by separate required `rangeMin`/`rangeMax` scalars.

### DP-2 — secondaryLabel scratchpad pollution
**Fix:** Two layers. (1) The `compare_datasets` sub-generator is the only path that prompts for secondary labels; schema description explicitly caps labels at 25 chars, "no numbers, no narration." (2) `sanitizeLabel()` post-process trims, strips trailing non-alphanumeric tails (`,`, `]`, spaces), collapses whitespace, and truncates to 40 chars. Other modes never emit a secondary label because their schemas don't include one — structurally impossible to pollute.

### DP-3 — Missing answer fields on fallback
**Fix:** POST-PROCESS-DERIVE. `targetAnswer` for read_frequency derived via `computeMode` / `computeLeastFrequent` on generated `dataPoints`. `targetAnswer` for compute_stats derived via `computeMedian` / `computeMode` / `computeNumericRange` based on generated `targetStat`. `comparisonAnswer` for compare_datasets derived via `deriveComparisonAnswer` — picks the label with the higher/lower median or more/less spread from the two generated datasets. Correct by construction — tutor/evaluator always gets a verifiable answer, never trusts Gemini for the numeric/string answer.

## Notes — Not flagged

- **Orchestrator eliminates the whole category of "multi-mode schema overload" bugs.** Each sub-generator sees only its own fields and prompt, so flash-lite can't cross-contaminate types (SP-3) and can't skip nullable fields (SP-14).
- **Label sanitizer is defense-in-depth**, not load-bearing. Schema constraint is primary; sanitizer catches any remaining edge cases.
- **`validation.typesFound: []` in API response** remains cosmetic — API validator scans `challenges[].type`, which is absent because this generator uses `challenges[].evalMode`. No student impact.
- **Stat panel gating**: confirmed via component code — `STATS_HIDDEN_MODES` correctly hides stats for the four β<4.5 modes; compute_stats and compare_datasets receive `showStatistics: true`.
