## Resolved Issues

### TF-3 ŌĆö make_ten word problem, frame, equation, and accepted answer all disagreed (CRITICAL ŌåÆ RESOLVED)
- **Was:** In double-frame mode the on-screen word problem ("There are 6 counters. How many
  more to make 10?") contradicted every other element. Frame pre-filled `targetCount` (10),
  equation read `10 + ___ = 20`, accepted answer `10`. A student following the word problem
  entered `4` and was marked wrong. The `wordNum(instruction, targetCount)` guard matched the
  "10" in "make 10" so it never rewrote the stale instruction (validator false-negative variant
  of SP-17).
- **Root cause:** Two sources of truth. The schema asked Gemini for an `instruction` string
  that duplicated facts the component already owns deterministically (`targetCount`,
  `totalCells`, complement). The component renders/scores purely from the numeric fields and
  ignores the instruction, so the LLM text could drift with no way to catch it ŌĆö the regex
  guard was a patch on the wrong layer.
- **Fix (SP-17 SCHEMA-SIMPLIFY):** Removed `instruction` from the Gemini schema entirely.
  It is now synthesized deterministically in post-process from the numeric fields via
  `buildInstruction(challenge, mode)` ŌĆö single source of truth, no guard can false-negative.
  Deleted the `wordNum` regex guard. Added explicit `addend1`/`addend2` fields so the one type
  whose content isn't derivable from `targetCount` alone (`add`) is also correct by construction.
- **Product decision:** make_ten is pedagogically "complement to 10" (catalog label, beta 3.5,
  and the tutor scaffold hardcode `= 10`). The make_ten eval mode is now pinned to single frame
  (before processing and again after manifest config overrides). A double-frame "make 20"
  complement, if ever wanted, is a *separate* eval mode (same schema, `mode: 'double'`), not a
  schema split ŌĆö once `instruction` is synthesized there is no remaining desync surface.
- **Files:** `service/math/gemini-ten-frame.ts` (schema, prompt, post-process),
  `primitives/visual-primitives/math/TenFrame.tsx` (added `addend1`/`addend2` to the challenge type).

## Notes
- Instructions are now uniform per challenge type rather than topic-flavored (accepted SP-17
