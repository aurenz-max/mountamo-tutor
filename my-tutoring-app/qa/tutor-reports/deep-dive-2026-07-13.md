## Findings

### TU-5 (HIGH) — block tap-to-explore + help forwards are non-silent → `tagged-sendtext-not-silent`. Fix in COMPONENT.

The 4 orchestration sends (`[DEEP_DIVE_START]`, `[ANSWER_CORRECT]`, `[ANSWER_INCORRECT]`, `[ALL_COMPLETE]`) correctly pass `{ silent: true }`. But **all 12 block→tutor forwards** are bare:

```
onAskTutor={(msg) => sendText(msg)}   // DeepDive.tsx:414,422,426,428,436,446,456,466,478,493,508,521
```

These carry the tap-to-explore directives (`[FACT_EXPLORE]`, `[COMPARE_EXPLORE]`, `[TIMELINE_EXPLORE]`, `[QUOTE_EXPLORE]`, `[TABLE_EXPLORE]`, `[PROSE_EXPLORE]`) **and** the graded-block help directive (`[STUDENT_HELP_REQUEST]`, built in `BlockTutorHelp` callers — e.g. [MultipleChoiceBlock.tsx:114](../src/components/lumina/primitives/visual-primitives/core/deep-dive/blocks/MultipleChoiceBlock.tsx#L114), [FillInBlankBlock.tsx:148](../src/components/lumina/primitives/visual-primitives/core/deep-dive/blocks/FillInBlankBlock.tsx#L148)).

**Mechanism** (`LuminaAIContext.sendText`, [line 785](../src/contexts/LuminaAIContext.tsx#L785)): a non-silent send (a) records the raw text as a `role: 'user'` conversation turn, (b) increments `conversationTurns`/`totalInteractions`, (c) sets `isAIResponding`, and (d) calls `ensureActive()` (claims tutor focus). So every tap/help forwards the literal tag directive — e.g. `[STUDENT_HELP_REQUEST] The student is stuck… The correct answer is "X". Guide them…` — into the transcript as a student turn.

**Blast radius (verified, honest):**
- On the **standard lesson path** the mounted surface is `CuratorCompanion` ([LessonScreen.tsx:219](../src/components/lumina/components/LessonScreen.tsx#L219)), which renders **assistant turns only** — so the raw directive/answer is **not visibly displayed** there. Impact on that path is metric pollution + spurious focus claims.
- On any **user-turn-rendering surface** (`AIHelper.tsx` [line 173-185](../src/components/lumina/components/AIHelper.tsx#L173), used by PracticeMode / KnowledgeCheck / Pulse) the same send would display the tag syntax **and the embedded correct answer** as a chat bubble — a latent visible answer leak.

**Why Tier 1 passed it:** the static analyzer scopes `componentFiles` to `DeepDive.tsx` only. The `_EXPLORE`/`STUDENT_HELP_REQUEST` tag literals live in the 15 block files and reach `sendText` through the `onAskTutor` indirection (a `msg` variable), so the scan sees neither the tags nor their silent-status. deep-dive's tutoring surface is distributed across the block tree — a known analyzer blind spot for this primitive.

**Fix (COMPONENT, when approved):** wrap every block forward silent —
`onAskTutor={(msg) => sendText(msg, { silent: true })}`. Caveat to confirm during the fix: `silent: true` also skips `ensureActive()`; verify the DeepDive tutor's activation path independently (the intro `[DEEP_DIVE_START]` is already silent, so activation does not depend on these taps).

## Verdict

Connection is intact and every `{{var}}` resolves — but the block-emitted sends violate the silent-trigger contract that the orchestration sends honor. **1 HIGH (TU-5).** No Tier-1 var/leak issues.
