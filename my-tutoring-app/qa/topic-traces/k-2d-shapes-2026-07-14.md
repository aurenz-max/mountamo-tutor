## Scope drops

### shape-tracer — students asked to trace a square/rectangle are given triangle paths
- **Chain:** objective "explain why a shape has a specific name based on how many sides" ✓ → intent "trace triangles and rectangles, counting sides" ✓ → data: 3 challenges, all with 3-point triangle tracePaths, two labeled square/rectangle
- **Broken link:** GENERATOR — instruction text and tracePath are generated independently; nothing derives the path from the named shape
- **Fix target:** `gemini-shape-tracer` (derive tracePath from targetShape in code, not from the LLM — LLM window / code structure rule) + ContentOracle: vertex count ↔ shape name
- **Severity:** teaches the WRONG shape-name/side-count mapping — the exact concept the objective targets. Breaks all grades, not just K.

### sorting-station — challenge 2 sorts by color for a shape-constancy objective
- **Chain:** objective "match shapes … different sizes or turned a different way" ✓ → intent "sort into bins based on shape category" ✓ → data c2 `sortingAttribute: "color"`, c3 size
- **Broken link:** GENERATOR — treats attribute variety as enrichment; for this objective color-sorting is off-task
- **Fix target:** `gemini-sorting-station` — bind sortingAttribute to the objective (recurring pattern: same drift in needs/wants and community-helpers traces)

### knowledge-check — text-described shapes are unanswerable
- Both categorization items describe square and rectangle with the same sentence; correct sorting is impossible from what the student sees. MCQ answers are readable in the option text itself.
- **Fix target:** shape questions must carry visual insets; PRE band-gate for option modality.
